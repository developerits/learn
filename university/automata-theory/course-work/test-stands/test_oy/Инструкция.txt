Параметры стенда:
n - разрядность формируемых операндов
mode - код, в котором выполняется операция
start_patern - начальное значение счётчика (по умолчанию все '0')
stop_patern - конечное значение счётчика   (по умолчанию все '1')	

1. Скопируйте файлы проекта вашего ОУ в папку MY_OY.
2. Добавьте их к проекту Project->Add Current File to Project.
3. Подключение ОУ к стенду выполните с помощью файла config.vhd, 
в котором вместо use entity work.Operation_device(behav1) 
укажите entity вашего ОУ и его архитектурное тело.
4. Выполните компиляцию проекта.
5. Откройте файл test_for_stand из папки MY_OY и запустите функциональное моделирование.

Примечание: 
1. Имена портов вашего ОУ должны совпадать с именами компонента ОУ стенда TEST_OY.
2. Ваше ОУ должно иметь параметр n(разрядность), значение которого не задано, то есть generic(n:integer);
3. Проверьте значение параметров  mode и n в файле TEST_OY, перед моделированием.
4. В случае нарушения русскоязычных комментариев в коде, рекомендуетя открыть файл в блокноте, 
после чего произвести следующие манипуляции Файл->сохранить как и выбрать кодировку UTF-8 или ANSI.
